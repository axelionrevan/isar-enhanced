# 使用限制

如你所知，Isar可以通过虚拟机在移动设备和桌面端工作，也可以在网页端工作。这两个平台非常不同，所以有不同的限制。

## 虚拟机（VM）限制

- 只有一个字符串的前1024字节（byte）可以用于前缀（prefix）where语句
- 对象的大小只能是16MB

## 网页端（Web）限制

因为Isar的网页端依赖于IndexedDB，所以会有更多的限制，但在你使用Isar时几乎感受不到。

- 不支持同步方法
- 目前`Isar.splitWords()`和`.matches()`过滤器还没有实现。
- 模式的改变不像在虚拟机中那样受到严格的检查，所以要注意遵守规则。
- 所有的数字类型都存储为double（js里唯一的数字类型），所以`@Size32`不会起到任何作用。
- 索引的表示方式不同，所以哈希索引不会使用更少的空间（它们的工作方式仍然相同）。
- `col.delete()`和`col.deleteAll()`能正常工作，但返回值不正确
- `col.clear()`不会重置自增值
- 不支持`NaN`作为一个值
